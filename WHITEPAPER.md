## Background and Context

### The Aging-in-Place Movement

The global population is aging at an unprecedented rate. By 2030, adults aged 65 and older will outnumber children under 5 for the first time in human history. Research consistently shows that the vast majority of older adults (approximately 90%) prefer to age at home rather than in institutional settings.

However, aging in place successfully requires:
- Access to accurate, relevant information
- Support systems for both individuals and caregivers
- Practical tools for assessment and planning
- Connection to community resources
- Evidence-informed guidance from trusted sources

### SageÉlan Foundation's Role

The SageÉlan Foundation was established to address these needs through a companionship-based, person-centered approach. The "Sage With You - Living in Place" program emphasizes:

1. **Dignity:** Respecting individual autonomy and choice
2. **Companionship:** Supporting rather than directing
3. **Evidence-Informed:** Grounding guidance in research and best practices
4. **Accessibility:** Ensuring services reach those who need them most
5. **Holistic Wellbeing:** Addressing physical, emotional, social, and practical needs

### Digital Transformation Imperative

Prior to this platform, the Foundation faced challenges in:
- Reaching geographically dispersed populations
- Providing 24/7 access to resources
- Scaling personalized assessments
- Tracking engagement and impact
- Demonstrating organizational capacity to grantmakers

The digital platform was conceived as a solution to these challenges while maintaining the Foundation's core values of warmth, accessibility, and person-centered support.

## Problem Statement

### Primary Challenges Addressed

#### 1. Information Access Barriers
- **Challenge:** Older adults and caregivers struggle to find reliable, comprehensive information about aging in place
- **Impact:** Delayed decision-making, preventable crises, unnecessary institutionalization
- **Solution:** Centralized, accessible resource library with evidence-based content

#### 2. Self-Assessment Difficulties
- **Challenge:** Individuals lack structured tools to evaluate their readiness and needs for aging at home
- **Impact:** Unrealistic expectations, inadequate planning, safety risks
- **Solution:** Interactive self-assessment tools with personalized recommendations

#### 3. Caregiver Support Gaps
- **Challenge:** Family caregivers are often unprepared and under-supported
- **Impact:** Caregiver burnout, relationship strain, compromised care quality
- **Solution:** Dedicated resources, planning tools, and educational content for caregivers

#### 4. Service Navigation Complexity
- **Challenge:** Fragmented service landscape is difficult to navigate
- **Impact:** Unmet needs, service underutilization, inefficient resource allocation
- **Solution:** Organized resource directory with planning dashboard functionality

#### 5. Digital Accessibility
- **Challenge:** Many aging-focused websites are not truly accessible to older adults
- **Impact:** Digital divide, exclusion of target population, reduced impact
- **Solution:** Universal design principles with adjustable accessibility features

#### 6. Organizational Visibility
- **Challenge:** Limited online presence reduces grantmaker confidence and partnership opportunities
- **Impact:** Funding challenges, missed collaborations, limited reach
- **Solution:** Professional, comprehensive website demonstrating capacity and impact

## Technical Architecture

### Technology Stack Overview

The platform is built using modern, proven technologies selected for performance, accessibility, and maintainability:

#### Frontend Framework: React 18.3.1
**Why React:**
- Component-based architecture enables reusability and maintainability
- Large ecosystem of accessibility-focused libraries
- Excellent performance for interactive features
- Strong community support and documentation
- Industry standard for complex web applications

#### Styling: Tailwind CSS v4.0
**Why Tailwind:**
- Utility-first approach enables rapid, consistent styling
- Built-in responsive design utilities
- Excellent accessibility features
- Easy customization for brand alignment
- Small production bundle size with purging

#### Routing: React Router v7 (Data Mode)
**Why React Router:**
- Robust client-side routing for single-page application
- Clean URLs enhance SEO and user experience
- Nested routing supports complex navigation structures
- Code-splitting capabilities improve performance
- Industry-leading documentation and support

#### Build Tool: Vite 6.3.5
**Why Vite:**
- Lightning-fast development server with hot module replacement
- Optimized production builds
- Native ES modules support
- Excellent developer experience
- Modern, actively maintained

#### Component Library: Custom + Radix UI
**Why Custom Components:**
- Full control over accessibility implementation
- Brand-aligned design system
- Optimized for target audience needs
- Radix UI primitives ensure accessibility fundamentals

#### Icon System: Lucide React
**Why Lucide:**
- Comprehensive icon set
- Consistent visual language
- Lightweight and tree-shakeable
- Accessible by default

### Architecture Patterns

#### Component-Based Architecture
```
/src/app/
├── App.tsx                    # Root component with password gate
├── routes.tsx                 # Route configuration
├── components/                # Reusable components
│   ├── ui/                   # Base UI components (buttons, inputs, cards)
│   ├── Navigation.tsx        # Site navigation
│   ├── Footer.tsx            # Site footer
│   ├── PasswordGate.tsx      # Staging access control
│   └── ...                   # Feature-specific components
├── pages/                     # Page components
│   ├── Home.tsx
│   ├── About.tsx
│   ├── Programs.tsx
│   ├── Resources.tsx
│   ├── Contact.tsx
│   └── auth/                 # Authentication pages
└── lib/                       # Utilities and helpers
    └── supabase-client.ts    # Mock authentication client
```

#### State Management Strategy
- **Local State:** React hooks (useState, useReducer) for component-level state
- **URL State:** React Router for navigation and filter state
- **Persistent State:** localStorage for user preferences and mock authentication
- **Future:** Supabase integration for real-time and database state

#### Responsive Design Approach
- **Mobile-First:** Base styles optimized for mobile devices
- **Breakpoints:** 
  - Mobile: < 768px
  - Tablet: 768px - 1024px
  - Desktop: > 1024px
- **Flexible Grid:** CSS Grid and Flexbox for adaptive layouts
- **Fluid Typography:** Responsive font sizing with accessibility overrides

## Accessibility and Universal Design

### Accessibility Commitment

The SageÉlan Foundation platform is built on the principle that accessibility is not a feature—it's a fundamental requirement. Given our target audience of older adults and caregivers, many of whom may experience vision, hearing, motor, or cognitive changes, universal design is embedded throughout.

### WCAG 2.1 Compliance

**Target Level:** AA (with AAA elements where feasible)

**Conformance Categories:**

#### Perceivable
✅ **Text Alternatives:** All images have descriptive alt text  
✅ **Adaptable Content:** Semantic HTML with proper heading hierarchy  
✅ **Distinguishable:** 4.5:1 contrast ratio minimum (7:1 for large text)  
✅ **Color Independence:** Information never conveyed by color alone  

#### Operable
✅ **Keyboard Accessible:** Full site navigation without mouse  
✅ **Enough Time:** No time limits on interactions  
✅ **Seizure Prevention:** No flashing content  
✅ **Navigable:** Clear focus indicators, skip links, breadcrumbs  

#### Understandable
✅ **Readable:** Plain language, adjustable text size  
✅ **Predictable:** Consistent navigation and behavior  
✅ **Input Assistance:** Clear labels, error prevention, helpful validation  

#### Robust
✅ **Compatible:** Valid HTML, ARIA landmarks, tested with assistive technology  

### Custom Accessibility Features

#### 1. Font Size Control
**Location:** Available on all pages  
**Functionality:**
- Small (90% of base)
- Default (100%)
- Large (125%)
- Extra Large (150%)

**Implementation:**
- Persists across sessions (localStorage)
- Affects all text elements proportionally
- Maintains layout integrity at all sizes

**User Benefit:** Accommodates vision changes without requiring browser zoom

#### 2. High Contrast Mode
**Location:** Available on all pages  
**Functionality:**
- Toggle high contrast color scheme
- Enhanced text/background contrast
- Stronger border and focus indicators

**Implementation:**
- Persists across sessions
- Meets WCAG AAA standards (7:1 minimum)
- Maintains brand recognition

**User Benefit:** Assists users with low vision or color blindness

#### 3. Keyboard Navigation
**Coverage:** 100% of interactive elements  
**Features:**
- Visible focus indicators
- Logical tab order
- Skip navigation links
- Keyboard shortcuts for common actions

**Testing:** Regular keyboard-only navigation testing

#### 4. Screen Reader Optimization
**Implementation:**
- Semantic HTML5 elements
- ARIA labels and landmarks
- Descriptive link text
- Form label associations
- Dynamic content announcements

**Testing:** Regular testing with:
- NVDA (Windows)
- JAWS (Windows)
- VoiceOver (Mac/iOS)
- TalkBack (Android)

### Cognitive Accessibility

#### Clear Information Architecture
- Maximum 3 levels of navigation
- Consistent menu structure
- Breadcrumb trails
- Clear page titles

#### Progressive Disclosure
- Complex information revealed gradually
- "Learn More" expandable sections

CREATE TABLE profiles (

  updated_at TIMESTAMP DEFAULT NOW()

CREATE TABLE saved_resources (

  created_at TIMESTAMP DEFAULT NOW()

CREATE TABLE assessment_results (

  recommendations JSONB,
